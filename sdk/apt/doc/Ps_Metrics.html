Introduction

Ps_Metrics calculates various QC metrics for SNPs genotyped by Affymetrix Genotyping Console (GTC) or Affymetrix Power Tools (APT) for Axiom arrays. Ps_Metrics takes standard outputs from GTC/APT and generates a data frame containing quality measurement metrics for all input SNPs. More details describing the use of Ps_Metrics and detailed explanations for each of the metrics reported can be found in the SNPolisher User Guide available on Affymetrix.com.

Quick Start

To run Ps_Metrics on posterior and calls files in directory ./input and generate output file metrics.txt: 

Ps_Metrics --posterior-file ./input/AxiomGT1.snp-posteriors.txt --call-file ./input/AxiomGT1.calls.txt --metrics-file metrics.txt

Details

The following parameters are required to run Ps_Metrics:

  • posterior-file - The file output by GTC/APT which contains posterior summaries of clusters for SNPs (usually named "AxiomGT1.snp-posteriors.txt").
  • metrics-file - The name of the output file. The user must supply a file name in order to save the output as a separate file, which is read in to other SNPolisher functions such as Ps_Classification.

In addition, one of the following two parameters is required:

  • call-file - The file output by GTC/APT which contains genotype calls for SNPs (usually named "AxiomGT1.calls.txt"). The call file may use either the labels "AA", "AB", "BB" and "NoCall", or the numeric labels 0 (AA), 1 (AB), 2 (BB) and -1 (NoCall).
  • chp-files - A text file containing a list of CHP files from which to take calls (one file path per line).

Note either the --call-file or the --chp-files parameter must be provided in order to run Ps_Metrics.
